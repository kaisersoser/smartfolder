# smartfolder 2.25 UX release notes

smartfolder 2.25 is a desktop UX refinement and trust release. It keeps the existing metadata-only analysis, preview-before-moving, no-overwrite apply behavior, restore history, saved profiles, and optional AI assistance while making the GUI calmer, more explicit, and easier to maintain.

## Highlights

- Organize, Preview, Completion, Activity, Rules, Rule Builder, and Settings now use the v2.25 design foundation.
- Preview shows real untouched-file reasons from the core planner and SQLite session store instead of inferred GUI copy.
- Preview supports paged data, reason breakdowns, row explanations, and a tree view.
- Completion and Activity use restore-oriented language and keep technical ids/paths collapsed by default.
- Rules and Rule Builder include the refined profile workspace, visual destination builder, simulation feedback, AI draft review, and AI rule-builder presentation.
- Command palette actions cover the main workflow shortcuts.
- Rendering code is split into screen modules for activity, organize, preview, rules, and settings.

## Validation

The release candidate passed:

```text
cargo fmt --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
cargo build -p smartfolder-gui --release
```

Additional release checks passed:

- Disposable-folder proof: analyze -> apply -> undo restored files to original locations.
- Human GUI click-through of the v2.25 workflow.

## Compatibility

This release does not change the safety model. smartfolder still scans metadata only, previews planned moves before organizing, never overwrites existing files, and records restore history before moving files.
